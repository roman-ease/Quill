## ツール概要

**MarkdownViewer** は、Markdown ファイルをリアルタイムでプレビューしながら編集できる、軽量なデスクトップアプリケーションです。

### 対象ユーザー

- Markdown を日常的に使用するエンジニア・ライター・ドキュメント担当者
- Obsidian / VS Code などのエコシステムと並行して使うユーザー
- 軽量かつスタンドアローンな Markdown ビューア兼エディタを求めるユーザー

### コンセプト

- **軽量・高速**: 大きなランタイムに依存せず、起動・動作が速いこと
- **スタンドアロン**: 別途サーバーやブラウザを立ち上げずに単体で完結すること
- **ネイティブ操作感**: OS のファイルダイアログ・クリップボード・ドラッグ&ドロップを活用すること
- **シンプルなUX**: 設定やカスタマイズは用意しつつ、デフォルトですぐ使えること

### 14. ファイル関連付けとシングルインスタンス制御

#### 14-1. ファイル関連付け

OS に対して以下の拡張子をこのアプリに関連付けてください。

| 拡張子 | 関連付け |
|--------|---------|
| `.md` | このアプリで開く（デフォルト関連付け） |
| `.markdown` | このアプリで開く（デフォルト関連付け） |

- ダブルクリック時にアプリが起動し、対象ファイルをタブとして開くこと
- 関連付けの登録・解除はインストール・アンインストール時に自動で行うこと
- OS のデフォルトアプリ設定から変更できる状態にすること（Windowsならレジストリ、macOSならInfo.plist、Linuxなら `.desktop` ファイル等、OSに適切な方法で登録）

#### 14-2. シングルインスタンス制御

**アプリは常に1つのプロセスのみ起動する**ことを原則とし、以下の挙動を実装してください。

| ケース | 挙動 |
|--------|------|
| アプリが起動していない状態でファイルをダブルクリック | アプリを起動し、そのファイルをタブとして開く |
| アプリが起動中の状態でファイルをダブルクリック | 既存ウィンドウにタブを追加してそのファイルを開く。ウィンドウをフォアグラウンドに移動する |
| アプリが起動中の状態で**既に開いているファイル**をダブルクリック | 新しいタブを開かず、既存タブに切り替える。ウィンドウをフォアグラウンドに移動する |
| 複数ファイルを同時に選択してダブルクリック / コマンドラインで複数ファイルを渡す | すべてのファイルを個別タブとして開く（既に開いているものはスキップ） |
| 前回プロセスがクラッシュしてゾンビ状態の場合 | プロセスの実在を確認し、実際に動作していなければ新しいプロセスとして起動する |

#### 14-3. インスタンス間通信の実装方針

実装方法は OS・フレームワークに委ねますが、以下の点を満たしてください。

- **プロセスの生存確認**: PIDの記録だけでなく、実際にプロセスが動作しているかを確認すること（ゾンビPID対策）
- **ファイルパスの受け渡し**: セカンダリ起動時（既にアプリが動いている状態）にファイルパスをプライマリプロセスへ確実に渡せること
- **応答待ちタイムアウト**: プライマリプロセスへの通知が一定時間内に確認できない場合は、新しいプロセスとして起動すること
- **CPU 負荷への配慮**: インスタンス監視のポーリング間隔は 1 秒以上を推奨（短すぎるとバックグラウンド時の CPU 消費が増大する）

> **実装例（参考）**: Named Pipe / Unix Domain Socket / 共有ファイル＋ロック / TCP ローカルポート など、OS・フレームワークで使いやすい IPC 手段を選んでください。

## 実装上のヒント（参考情報）

> ここに記載した情報は「このプロジェクトで経験した知見」です。
> 別端末・別スタックでは異なる実装が最適な場合があります。Claude Code の判断に委ねます。

- **Mermaid.js / KaTeX は遅延ロードを推奨**: 初回使用時に動的ロードすることで起動を高速化できます
- **同期スクロールのループ防止**: `scrollSource` フラグとタイムアウトを組み合わせてループを抑制してください
- **ファイル監視は全タブ一括で**: タブごとに個別インターバルを立てると多重起動時にリソース消費が増えます
- **画像は Base64 変換してプレビュー**: ローカルファイルのパスをそのまま `<img src>` に渡しても表示できない環境があるため、Base64 変換を検討してください
- **HTML エクスポート時の CSS 変数**: CSS 変数をそのまま埋め込むと別ブラウザで表示が崩れます。`getComputedStyle()` 等で実値に展開してください
- **シングルインスタンス制御の落とし穴**: PID のみで「起動中かどうか」を判定すると、クラッシュ後に残ったゾンビ PID を誤検知して二重起動を拒否することがあります。PID に加えてプロセス名やソケット接続の確認を組み合わせてください
- **ファイル関連付け後の CWD**: OS のファイル関連付けからアプリを起動すると、カレントディレクトリ（CWD）がアプリのインストールディレクトリ以外になる場合があります。リソースファイルの参照には CWD ではなく EXE / エントリポイントの絶対パスを使ってください
